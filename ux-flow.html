UX Flow Overview

Harvey Events User Journey

Presentation view of the complete prototype flow. This page summarizes the main screens, decision points, and possible routes so the product can be explained quickly to the team, stakeholders, or client.

Start: Welcome
Branches: login or registration
Key decision: valid or invalid code
Final result: products added + installation choice
Post-flow: main app, secondary GPS/Land options
Step 1

Welcome

Initial screen that introduces the benefit and offers two access paths.

Goal: present the proposal and open the flow.
Exits: Log In, Create account.
Step 2A

Sign In

Existing user signs in with email and password to continue redeeming the product.

Entry: from Welcome.
Exit: sent to Promo Code.
Step 2B

Register

New user creates an account before continuing with the promotional flow.

Entry: from Welcome.
Exit: sent to Promo Code.
Decision

Promo Code

Central point of the journey. The system evaluates the entered code.

Valid code: `MAPA2026`.
Branches: success or invalid code.
Step 3

Success

Confirmation that products were added and selection of the installation tutorial.

Result: product added successfully.
Exits: main TwoNav App, secondary GPS/Land.
Support: email with the information.
Error State

Invalid Code

Alternate state when the code is not valid and the user needs to try again.

Result: handled error.
Exit: back to Promo Code.
Main Post Flow

App TwoNav

Priority tutorial for installing the product in the Android/iOS app.

Entry: main CTA from Success.
Includes: step 0 to install the app.
Secondary Post Flow

GPS TwoNav

Secondary access for users with a TwoNav GPS device.

Entry: discreet link from Success.
Role: support for dedicated device.
Secondary Post Flow

Land

Secondary access for using the product from a Windows/Mac computer.

Entry: discreet link from Success.
Role: support for planning on a computer.
