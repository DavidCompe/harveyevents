UX Flow Overview

Harvey Events User Journey

Vista de presentacion del flujo completo del prototipo. La pagina resume las pantallas principales, los puntos de decision y las rutas posibles para explicar el producto de forma rapida a equipo, stakeholders o cliente.

Inicio: Welcome
Ramas: login o registro
Decision clave: codigo valido o invalido
Resultado final: productos anadidos + instalacion
Step 1

Welcome

Pantalla inicial que introduce el beneficio y ofrece dos caminos de acceso.

Objetivo: presentar la propuesta y abrir el flujo.
Salidas: Log In, Create account.
Step 2A

Sign In

Usuario existente accede con email y password para continuar con el canje.

Entrada: desde Welcome.
Salida: envio a Promocode.
Step 2B

Register

Usuario nuevo crea cuenta antes de seguir con el flujo promocional.

Entrada: desde Welcome.
Salida: envio a Promocode.
Decision

Promocode

Punto central del journey. El sistema evalua el codigo introducido.

Codigo valido: `MAPA2026`.
Ramas: success o invalid code.
Step 3

Success

Confirmacion de productos anadidos a la cuenta y acceso al siguiente paso.

Resultado: alta correcta del producto.
Salidas: instalacion o volver al inicio.
Error State

Invalid Code

Estado alternativo cuando el codigo no es valido y el usuario debe reintentarlo.

Resultado: error controlado.
Salida: volver a Promocode.
Post Flow

Install Instructions

Pantalla de apoyo que explica el siguiente paso una vez recibido el producto.

Entrada: desde Success.
Rol: soporte post-canjes.
